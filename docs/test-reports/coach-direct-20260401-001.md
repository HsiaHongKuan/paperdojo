### Design Observations

1. **No interests.toml fallback messaging.** When interests.toml is absent, the coach silently uses defaults. Consider having the coach mention: "I don't see a profile yet — I'll calibrate to a balanced level. Run /setup to customize." Improves discoverability.

2. **Outcome classification threshold is undefined.** The user arrived at the conceptual solution but never said "belief propagation" or "boundary MPS" by name. The SKILL.md doesn't define when "solved" vs "revealed" applies. Need guidance: solved = user identifies the core mechanism, even without naming the specific technique.

3. **Hint ladder is single-track.** The "local messages between neighbors" hint works for users who know graphical models. If the user doesn't make the BP connection, there's no specified fallback hint strategy. The skill should suggest a two-tier approach: abstract nudge first, concrete nudge if needed.

4. **Step 2 suitability message is ambiguous.** The coach said "This paper is a great fit" to the user, but the spec implies the suitability check is internal. For unsuitable papers, the spec says "tell the user honestly." For suitable papers, it just says "proceed." An explicit "say nothing to the user" instruction for suitable papers would prevent inconsistency.

## Suggestions

1. **High impact: Define "solved" vs "revealed" threshold.** Add to spec: "solved" means the user identified the core conceptual mechanism (not necessarily by name). "revealed" means the user asked to see the solution before reaching the core idea.

2. **High impact: Add fallback hint strategy.** When a hint doesn't land, the skill should specify an escalation path — more concrete hints at each level, up to directly naming the conceptual area without naming the technique.

3. **Medium impact: Add missing-profile messaging.** When interests.toml doesn't exist, briefly note that defaults are being used and suggest /setup.

4. **Medium impact: Clarify Step 2 user-facing output.** For suitable papers: proceed silently (no announcement). For unsuitable: tell user and suggest alternatives. Make this explicit.

5. **Low impact: Suggest follow-up resources in Step 6.** The warm closing could point to related concepts or techniques the user should explore, beyond just "concepts to revisit."
