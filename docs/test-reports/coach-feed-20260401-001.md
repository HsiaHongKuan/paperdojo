### Design Observations

1. **Feed-to-coach transition is underspecified.** The spec says "build on and deepen" the feed [d] details, but doesn't say how to reference specific content from the Overview box. The coach said "Since you already looked at the overview" but didn't quote or extend specific points. A real implementation might re-derive from scratch.

2. **No "teach me the prerequisite" mode.** When the user lacks a fundamental concept (operator algebras), the Socratic loop can't bridge the gap. The coach can only hint progressively closer. Adding a "pause and explain" option would help cross-disciplinary users.

3. **Coaching preferences work well for this case.** Guided difficulty prevented the user from spinning for too long. Pointing style gave directional hints rather than open questions. The combination is appropriate for a user out of their depth.

4. **No mid-session exit or resume.** The spec has no provision for "save and continue later." The session JSON could support this but the skill doesn't mention it.

5. **Round count is implicit.** The spec doesn't define when to intervene more aggressively if the user isn't making progress. With "guided" difficulty, the coach should arguably step in earlier — but the boundary between "letting them think" and "letting them flounder" is judgment-dependent.

## Suggestions

1. **High impact: Add concrete feed [d] content carryover.** When entering from feed with details_viewed=true, the coach should explicitly reference and extend specific points from the Overview box rather than re-presenting from scratch. This makes the transition feel connected.

2. **High impact: Add a "teach me" escape hatch in Step 4.** When the user explicitly says they lack a prerequisite concept, allow a brief didactic sidebar before resuming Socratic mode. The coaching format assumes a minimum shared vocabulary that may not exist for cross-disciplinary users.

3. **Medium impact: Add mid-session save/resume.** Allow "I need to pause" during Step 4, saving session state so the user can return later. The JSON format already supports this.

4. **Medium impact: Post-reveal difficulty self-assessment.** After Step 5, ask "how challenging was this, 1-5?" to calibrate future coaching. Currently coaching prefs are static in interests.toml.

5. **Low impact: Define "guided" intervention threshold.** Specify that in guided mode, if the user's last 2 responses show no new progress, the coach should offer a more direct hint. This prevents the "spinning wheels" experience Alex had in Rounds 3-4.
